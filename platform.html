Platform

One modular foundation, built to support more than batteries

ReCore's platform is a shared foundation of sensor nodes, hubs, and cloud connectivity. Sentinel is the first product built on it, not the only one it is meant for.

Why Modular

One architecture, many possible jobs

Most monitoring products are purpose-built: one sensor, one box, one job. ReCore is built the other way. The Power Node, the Sentinel Hub, the local dashboard, and the cloud upload path are all pieces of a shared architecture, not a one-off product. That means the engineering work behind Sentinel, from LoRa communication to local-first reliability to secure cloud upload, is reusable for other kinds of monitoring as they are developed.

Today, that architecture is proven through a single working sensor type: battery voltage and temperature. The platform's value shows up over time, as additional sensor types are added to nodes that already know how to talk to a Hub, and Hubs that already know how to talk to the cloud.

How Data Moves

Transport options, matched to the job

Working now

LoRa

Long-range, low-power wireless used between Power Nodes and the Sentinel Hub. Built for sensors that need to reach across a yard, pasture, or shop without draining a battery to stay connected.

Working now

Wi-Fi

Used by the Sentinel Hub to reach your local network and, when available, upload telemetry to the ReCore cloud over HTTPS.

Planned

Ethernet

A wired connection option for the Hub in shops, equipment yards, or other fixed installations where a wired network is more practical than Wi-Fi.

Planned

Cellular

Being considered primarily as a connectivity option for the Sentinel Hub in locations without Wi-Fi, with direct-cellular nodes possible for isolated or mobile assets.
