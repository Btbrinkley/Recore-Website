Development & Engineering

Built in the open, tested on real equipment

ReCore and Sentinel are actively being designed and field-tested. This page tracks where things genuinely stand, updated as milestones are reached rather than as marketing copy.

How We're Building This

Build a working piece, test it for real, refine it

ReCore's development process is deliberately unglamorous: build a small working piece of the system, put it on real equipment, see what breaks or gets annoying, and refine it before adding the next piece. That is how the current Sentinel prototype came together, one working link at a time, from a Power Node reading a battery to a Hub relaying that reading to a phone or laptop on the same network.

We test outside the lab bench on purpose. Real batteries, real weather, and real equipment expose problems that a bench test never will, whether that is a LoRa signal getting blocked by a metal shop wall or a Hub losing Wi-Fi mid-afternoon. Field testing is how we find those problems while the stakes are still low.

Development Journal

Milestones so far

Working

Power Node prototype

Built and tested a battery-powered sensor node that reads voltage and temperature and transmits readings over LoRa.

Working

Sentinel Hub & local dashboard

The Hub receives LoRa packets from a Power Node and serves a live dashboard on the local network, viewable without any internet connection.

Working

Cloud telemetry pipeline

Connected the Hub to an HTTPS Cloud Function that writes incoming readings into Firestore. The first live Sentinel telemetry record successfully reached Firestore during this phase, confirming the full Node-to-cloud path works end to end.

In progress

Field testing

Running the complete Power Node to LoRa to Hub to local dashboard to Firestore path on real equipment, outside a lab bench, to find what needs to be more rugged before it becomes a product.

Planned

Historical charts, alerts, accounts, cellular, production hardware

The next phase of work: cloud-side history and alerting, account login and device management, optional cellular connectivity, and production-ready hardware. None of this has started shipping yet.
