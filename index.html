What Is ReCore

A modular platform, proven one working system at a time.

Most monitoring products are built around a single sensor and a single job. ReCore starts from the opposite direction. We are building a shared foundation of sensor nodes, a local hub, and cloud connectivity, so the same core system can support many different kinds of monitoring as new sensor types are added.

That foundation is proven today through Sentinel, our first product line. Sentinel exists to solve a problem equipment owners run into constantly: batteries that quietly die in vehicles, tractors, and equipment that no one happens to be watching at the time.

First Product Line

Sentinel: battery and equipment monitoring, built for real conditions

Sentinel starts with the fundamentals: battery voltage and temperature. A small Power Node attaches to the battery or equipment being monitored and reports its readings over LoRa, a long-range, low-power wireless protocol, to a central Sentinel Hub. The Hub keeps a local dashboard running on your own network, and, when it has an internet connection, uploads the same telemetry to the ReCore cloud.

Why It Matters

Built around how equipment actually gets used

Local-first reliability

The Sentinel Hub keeps monitoring and displaying data on its own local dashboard even if Wi-Fi or internet access drops. Cloud connectivity adds visibility; it is not required for the system to keep working.

Long-range, low-power sensing

LoRa lets sensor nodes reach a hub from across a yard, pasture, or shop without relying on Wi-Fi range or draining a battery to stay connected.

Designed to grow

The node-and-hub architecture is not locked to batteries. It is built so additional sensor types can be added to the same platform as they are developed.
